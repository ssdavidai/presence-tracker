# Storage Structure

All data is stored locally under `data/`. Nothing is sent to external services.

```
data/
├── chunks/          # Daily JSONL files — one per day, 96 lines each
│   ├── 2026-03-13.jsonl
│   ├── 2026-03-14.jsonl
│   └── ...
├── summary/         # Rolling stats (7-day, 30-day aggregates)
│   ├── rolling_7d.json
│   └── rolling_30d.json
├── dashboard/       # Daily HTML dashboards
│   ├── 2026-03-13.html
│   ├── 2026-03-14.html
│   └── ...
└── models/          # Trained ML model files
    ├── anomaly_detector.pkl
    ├── recovery_predictor.pkl
    └── focus_clusters.pkl
```

---

## data/chunks/

The primary data store. One JSONL file per ingested day. Files are named `YYYY-MM-DD.jsonl`.

**Created by:** `DailyIngestionWorkflow` (23:45) or manually via `python3 scripts/run_daily.py`.

**Size:** ~150KB per day (96 windows × ~1.5KB each).

**Retention:** No automatic cleanup. Files accumulate indefinitely.

---

## data/summary/

Rolling aggregate stats maintained by `engine/store.py` after each daily ingestion.

`rolling_7d.json` — averages and standard deviations for the past 7 days. Used by anomaly detection for baseline comparison.

`rolling_30d.json` — 30-day aggregates. Used by CDI calculation and the ML feature vectors.

---

## data/dashboard/

Static HTML files generated by `analysis/dashboard.py`. One per ingested day.

No server required — open directly in a browser, or serve with:

```bash
cd data/dashboard
python3 -m http.server 8888
# → http://localhost:8888/2026-03-14.html
```

---

## data/models/

scikit-learn model files serialised with `joblib`. Recreatable at any time by rerunning `scripts/train_model.py`.

**Do not commit these to git** — they contain personal behavioural data. The `.gitignore` excludes `data/models/*.pkl` and `data/chunks/*.jsonl`.
